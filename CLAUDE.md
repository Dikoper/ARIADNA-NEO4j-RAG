# CLAUDE.md — операционные правила агентов проекта «Ариадна»

Проект: GraphRAG-карта знаний R&D горно-металлургической отрасли
(Neo4j + локальные LLM семейства Qwen через Ollama + Claude API для ответов;
извлечение — qwen3.5:35b-a3b, откат qwen3.5:9b; эмбеддинги — qwen3-embedding:0.6b;
vLLM запаркован). Дедлайн сдачи: **4 июля 2026, 23:59**.

## Порядок чтения контекста (для любого агента)

1. Этот файл (загружается автоматически).
2. `docs/dev/TASK.md` — задание, онтология, 4 приёмочных запроса жюри.
3. `docs/dev/ARCHITECTURE.md` — схема модулей, потоки данных, инварианты системы.
4. `docs/dev/modules/<модуль>.md` — паспорт твоего модуля (у мета-модулей `scripts`
   и `docs` паспорта нет — контекст даёт постановка задачи от оркестратора).
5. `src/ariadna/contracts.py` — контракты данных (единственный источник истины).
6. `docs/dev/CONVENTIONS.md` — стандарты кода, пре-комментариев и логирования.
7. `docs/dev/worklogs/<модуль>.md` — история работ по твоему модулю.

Не загружай весь проект: твой рабочий пакет — CLAUDE.md + паспорт модуля + contracts.py
+ задача из STATUS.md + worklog модуля.

## Доска задач — STATUS.md (корень)

Словарь состояний: `📋 беклог · 🚧 в работе · 🧪 на тестировании · ✅ готово ·
⛔ заблокировано · 🔥 регрессия · 💤 отложено`

- Берёшь задачу → сразу переведи её 📋→🚧 (захват, защита от коллизий).
  Закончил код → 🚧→🧪.
- Правь ТОЛЬКО свою строку в STATUS.md. Чужие строки не трогать.
- Любое изменение кода = обновление STATUS.md + запись в worklog **в том же пакете
  изменений**.

## Worklog

`docs/dev/worklogs/<модуль>.md`, запись ≤ 8 строк по шаблону (см. шапку файла):
дата/агент/ID задачи + **Сделано / Решения / Проблемы / Открыто**.

## Лестница эскалации

1. Две самостоятельные попытки решить проблему.
2. Не вышло, или решение требует изменить contracts.py / архитектуру / 🔒-код →
   пиши в worklog блок `⛔ ЭСКАЛАЦИЯ:` (симптом, что пробовал, гипотезы), переведи
   задачу в ⛔ в STATUS.md и останавливайся.
3. `contracts.py` и функции с уровнем 🔒 меняет только solver/оркестратор.

## Git — строгая политика (решение PM)

- Исполнители НЕ выполняют `git commit` и НИКОГДА не выполняют `git push`.
- Коммитит только главный оркестратор и только после вердикта APPROVE от reviewer.
- `git push` — только PM вручную или по его явной команде.

## Код, тесты, логи

- Стандарты кода и пре-комментариев — `docs/dev/CONVENTIONS.md` (обязательны).
- Модуль ≤ ~350 строк; больше — сигнал к декомпозиции (обсуди через worklog).
- Логи — JSON Lines в `logs/pipeline/<run_id>.jsonl`; обязательные поля:
  `ts, run_id, module, stage, doc_id, level, event, detail`.
- Коды ошибок — реестр `docs/dev/ERRORS.md`; новый путь ошибки = новый код в реестре
  тем же пакетом изменений.
- Перед завершением задачи прогони: `python scripts/lint_precomments.py`
  и `python -m pytest tests/<модуль>/ -q`.
