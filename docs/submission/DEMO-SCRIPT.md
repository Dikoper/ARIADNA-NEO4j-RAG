# Сценарий видео-демо «Ариадна» (~5–7 минут)

Аудитория — жюри хакатона, нетехнический и технический состав. Реплики — простыми
словами, без внутреннего жаргона проекта (не говорить «GraphRAG», «эмбеддинги»,
«contracts.py» и т.п. — только по делу и понятно).

## Подготовка перед записью (сделать заранее, не в кадре)

1. `dc up -d` (алиас на `docker compose --env-file .env -f deploy/docker-compose.yml`),
   дождаться `healthy` у `ariadna_neo4j` и `ariadna_ollama` (`watch dc ps`).
2. Убедиться, что кэш ответов прогрет: файл `data/processed/answer_cache.json`
   существует и содержит все 4 вопроса (иначе первый клик по пресету в кадре уйдёт
   в живой синтез на 1–8 минут — испортит тайминг записи).
3. Поднять UI заранее и открыть его в браузере **до** начала записи:
   `.venv/bin/python -m streamlit run ui/app.py` → `http://localhost:8501`.
   Первый рендер Streamlit после холодного старта может тормозить — прогреть один клик
   вхолостую до записи.
4. Открыть вторую вкладку браузера на `http://localhost:7474` (Neo4j Browser),
   авторизоваться заранее (логин `neo4j`, пароль из `.env`) — чтобы такт (е) не
   тратил время на логин в кадре.
5. Разрешение экрана: 1920×1080, браузер в окне (не полноэкранный tiling) — таблицы
   и подграф в Streamlit шире на маленьком экране обрезаются горизонтальным скроллом.
6. Закрыть лишние вкладки/уведомления ОС — в кадре только браузер.
7. Прогнать один раз весь сценарий без записи — проверить тайминг, особенно такт (в).

## Потакт

### (а) 0:00–0:30 — проблема и решение

**Текст:** «У Норникеля тысячи страниц научных отчётов, статей и докладов по
горно-металлургическим технологиям. Найти в них ответ на конкретный вопрос —
человеку нужны часы. Мы сделали систему, которая читает весь этот архив, строит
из него карту знаний — кто, что и с какими цифрами исследовал — и отвечает на
вопрос за секунды, с точной ссылкой на источник вплоть до абзаца документа.»

Показать: главный экран UI (вкладка «Чат»), 4 кнопки-пресета вопросов жюри.

### (б) 0:30–2:30 — вопросы Q1–Q3, провенанс и подграф

**Текст:** «Вот реальные вопросы жюри хакатона.» Нажать пресет №1 (обессоливание
воды). Ответ приходит мгновенно (из кэша).

**Показать:**
- Текст ответа.
- Блок цитат — раскрыть 1–2 цитаты, показать: «Каждое утверждение в ответе — это
  не выдумка модели, а ссылка на конкретный кусок конкретного документа. Можно
  провалиться до первоисточника.»
- Подграф ответа (вкладка/блок с графом): «А вот как это выглядит в виде связей:
  материал → процесс → оборудование → результат.»

Повторить коротко для пресета №2 (циркуляция католита) — сделать акцент на
географии: «Система отдельно помечает, где это отечественная практика, а где —
мировая».

Пресет №3 (Au/Ag/МПГ штейн-шлак) — показать подграф с **красным ребром** (если
в подграфе конкретного ответа есть contradicts — заранее проверить на прогоне
без записи, какой из ответов его показывает; если ни один — показать красное
ребро отдельно на карте пробелов/произвольном узле в такте (г) вместо этого
момента и явно сказать: «когда в источниках есть противоречащие друг другу
данные — по одному эксперименту получили один результат, по другому другой —
система не прячет это, а подсвечивает связь красным прямо в графе»).

### (в) 2:30–3:30 — Q4, звёздный момент: честный «пробел»

**Текст:** «А вот четвёртый вопрос жюри — про закачку шахтных вод в глубокие
горизонты.» Нажать пресет №4.

**Показать:** ответ, где система прямо говорит, что по этой формулировке в
корпусе нет прямых экспериментов/публикаций, и предлагает смежные, близкие темы
(захоронение вод, поглощающие горизонты) вместо того, чтобы придумать
несуществующий ответ.

**Текст:** «Это специально. Многие системы в такой ситуации начинают
галлюцинировать — придумывать правдоподобный, но ложный ответ. Наша система
устроена иначе: если фактов нет, она честно об этом говорит и предлагает
ближайшую альтернативу. И это не единичный случай — у нас есть отдельный
инструмент, который системно ищет такие пробелы в знаниях.» → переход к карте
пробелов.

### (г) 3:30–4:45 — карта пробелов

Перейти на вкладку «Карта пробелов».

**Текст:** «Вот карта пробелов — таблица, где по строкам материалы, по столбцам

выщелачивание": именно эту комбинацию называет задание хакатона как пример
пробела, и наша система находит её первой же по значимости, без ручной
подгонки.»

Показать блоки «только в отечественной практике» (44 темы) и «только в
зарубежной практике» (225 тем).

**Текст:** «А это — темы, которые встречаются только в российских источниках или
только в зарубежных. Это прямой ответ на вопрос "где у нас нет собственной
практики, а весь опыт — импортный", и наоборот.»

### (д) 4:45–5:15 — фильтры гео/год

Вернуться на вкладку «Чат», открыть сайдбар.

**Текст:** «В сайдбаре есть фильтр по географии — отечественная практика,
зарубежная, или всё вместе — и по году публикации. Например, можно оставить
только источники за последние 5 лет.» Показать переключение фильтра года,
изменение списка видимых цитат.

### (е) 5:15–6:15 — ручная правка эксперта (У-4)

**Текст:** «В задании было требование — эксперт должен уметь вручную поправить
факт в базе, если он не согласен с автоизвлечением. У нас это работает прямо
через штатный интерфейс Neo4j.»

Переключиться на вкладку Neo4j Browser (`http://localhost:7474`, уже открытую).

**Готовый Cypher-запрос** (реальная сущность из текущего графа — материал
«медно-никелевая руда», ~13 упоминаний, confidence 0.9 по умолчанию):

```cypher
MATCH (e:Entity {id: "material:medno-nikelevaya-ruda"})
RETURN e.id, e.name, e.confidence, e.updated_at, e.edited_by, e.n_mentions;
```

Показать результат — узел со свойствами `confidence`, `updated_at`, `edited_by`
(пусто = ещё не редактировалось экспертом).

**Текст:** «Допустим, эксперт проверил этот факт и хочет повысить доверие к
нему и оставить пометку, что правил лично.» Выполнить:

```cypher
MATCH (e:Entity {id: "material:medno-nikelevaya-ruda"})
SET e.confidence = 0.98,
    e.edited_by = "expert_ivanov",
    e.updated_at = date().toString()
RETURN e.id, e.confidence, e.updated_at, e.edited_by;
```

**Текст:** «Готово — теперь у этого факта другая уверенность, есть дата
правки и имя эксперта. Всё это видно и в интерфейсе, и при любом следующем
ответе системы, который сошлётся на этот факт.»

*(Если для содержательности хочется показать конкретную связь, а не только
узел — можно дополнительно выполнить `MATCH (e:Entity {id:
"material:medno-nikelevaya-ruda"})-[r]-() RETURN e, r LIMIT 25` и открыть
любое ребро оттуда так же через `SET r.confidence = ..., r.edited_by = ...`.)*

## Резервные варианты (если что-то в кадре пойдёт не так)

- Если Streamlit подвис/белый экран — не паниковать в кадре, переключиться на
  заранее подготовленный скриншот/вторую вкладку с тем же состоянием (держать
  запасную вкладку открытой на нужном экране).
- Если live-Cypher в такте (е) выдал неожиданный результат (сущности не
  оказалось — например, граф пересобирался после проверки) — заранее (на
  прогоне без записи) подтвердить, что `material:medno-nikelevaya-ruda`
  существует; при расхождении — выполнить `MATCH (e:Entity) WHERE e.name
  CONTAINS "медно-никел" RETURN e.id, e.name LIMIT 5` в кадре и выбрать
  первый попавшийся `id` вместо жёстко прописанного.
