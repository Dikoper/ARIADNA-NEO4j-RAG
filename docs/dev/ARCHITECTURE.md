# ARCHITECTURE.md — схема модулей, потоки данных, инварианты

Все структуры данных между модулями — pydantic-модели из `src/ariadna/contracts.py`
(в скобках ниже). Менять контракты может только solver/оркестратор.

Онтология (фиксирована заданием, enum-ы в contracts.py):
- **8 типов сущностей:** Material, Process, Equipment, Property, Experiment,
  Publication, Expert, Facility.
- **6 типов связей:** uses_material, operates_at_condition, produces_output,
  described_in, validated_by, contradicts.

«TechSolution» ниже — НЕ отдельный EntityType, а неформальное имя второго хаба:
узлы Process с ролью технического решения.

## Модули: вход → выход (контракты)

| Модуль | Вход | Выход | Ключевые контракты |
|---|---|---|---|
| `ingest` | файлы `data/` | JSONL: тексты, мета, чанки | DocumentMeta, DocumentText, Chunk |
| `extraction` | Chunk | сущности/связи/числа | ExtractionResult, Entity, Relation, NumericConstraint |
| `graph` | ExtractionResult, Chunk+вектор | граф в Neo4j; Cypher-шаблоны | GraphNode, GraphEdge, Provenance |
| `search` | вопрос (строка) | Answer с цитатами + подграф | QueryIntent, Citation, Answer, Contradiction |
| `analytics` | граф | GapReport, LitReview, рекомендации, экспорт | GapReport, ReviewSection, ComparisonRow, Recommendation |
| `ui` | Answer, GapReport, LitReview | экраны Streamlit | — (только читает контракты) |
| `api` | HTTP | JSON поверх search/analytics | те же |
| `deploy` | — | docker-compose, env | — |

## Инварианты системы (нарушение = REJECT на ревью)

1. `contracts.py` — единственный источник истины о структурах данных; меняет только
   solver/оркестратор.
2. В Neo4j пишет только `graph` (loader). Остальные модули граф только читают.
3. Числовые значения и единицы измерения извлекаются ТОЛЬКО правилами
   (`extraction/rules`), не LLM. LLM может лишь указать, где искать.
4. Свободный text2cypher запрещён: только шаблоны из `graph` со слотами,
   которые заполняет роутер.
5. Каждый факт в графе несёт provenance до чанка + поля `source`, `updated_at`,
   `confidence` (У-4).
6. Ответ пользователю — всегда с цитатами (Citation). Нет источника — честное
   «в корпусе не найдено» (+ это вход для карты пробелов).
7. Логи — JSON Lines с `run_id`/`doc_id`; ошибки — с кодом из `docs/dev/ERRORS.md`
   и контекстом, достаточным для воспроизведения.

## Точка деградации

Чек-пойнт на ~10–12 часу разработки: если сквозной путь вопрос→граф→ответ не работает —
остаёмся на ingest + векторный слой + UI (RAG-first) с лёгким графом «документ—сущность»
для визуализации. Решение принимает PM.
