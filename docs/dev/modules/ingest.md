# Паспорт модуля: ingest

**Назначение:** конвертация документов корпуса (PDF/DOCX/PPTX/DOC) в нормализованный
текст с метаданными и чанкинг. Первый этап конвейера.

**Вход:** файлы `data/` (пути и папки-источники). Ядро: Обзоры + Статьи + Доклады.
**Выход:** JSONL-файлы в `data/processed/`: `meta.jsonl` (DocumentMeta),
`texts.jsonl` (DocumentText), `chunks.jsonl` (Chunk, без эмбеддингов).

**Контракты:** DocumentMeta, DocumentText, Chunk (`contracts.py`).

**Зависимости:** `pdftotext` (установлен в системе) / PyMuPDF, python-docx;
для PPTX — python-pptx. Выбор — за module-dev, критерий: скорость и полнота текста.

**Не входит в зону ответственности:** эмбеддинги (search), извлечение сущностей
(extraction), запись в Neo4j (graph), OCR сканов (вне скоупа хакатона).

**Известные ограничения:**
- PDF корпуса имеют текстовый слой (проверено выборкой); файлы без слоя — в skip-лист
  с логом INGEST-002, не чинить.
- Журнальные PDF содержат колонтитулы — чистить ДО чанкинга (иначе мусор в эмбеддингах).
- Метаданные (авторы, год) извлекать по возможности: имя файла, свойства документа,
  первая страница; отсутствие — не ошибка (поля Optional).
- Чанкинг: сохранять границы предложений; размер/перекрытие — именованные константы.
