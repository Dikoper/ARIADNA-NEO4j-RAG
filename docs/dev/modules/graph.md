# Паспорт модуля: graph

**Назначение:** всё, что пишет в Neo4j и описывает его схему: загрузка двойного графа
(лексический Document→Chunk с vector index + сущностный по онтологии), дедупликация
сущностей по синонимам, Cypher-шаблоны для роутера, семантический фасад.
ЕДИНСТВЕННЫЙ модуль с правом записи в Neo4j (инвариант №2).

**Вход:** `data/processed/*.jsonl` (Chunk с эмбеддингами, ExtractionResult).
**Выход:** наполненный Neo4j; библиотека Cypher-шаблонов с именованными слотами;
экспорт JSON-LD (A-18).

**Контракты:** GraphNode, GraphEdge, Provenance (`contracts.py`).

**Зависимости:** драйвер `neo4j`; подключение из `.env` (NEO4J_URI/USER/PASSWORD);
`ontology/` — OWL-файл и словарь синонимов RU/EN (тоже зона этого модуля, A-06).

**Не входит в зону ответственности:** вычисление эмбеддингов (search), выбор шаблона
и заполнение слотов (search/router), UI-визуализация.

**Известные ограничения:**
- Каждый узел/связь — с Provenance (doc_id, chunk_id, confidence, updated_at,
  edited_by — поля У-4). Констрейнты уникальности по id — до загрузки.
- Хабы онтологии: Experiment и «TechSolution» (НЕ отдельный EntityType, а узлы
  с label Process и ролью технического решения — см. ARCHITECTURE.md) —
  3 из 4 эталонных запросов про технические решения.
- Связь contradicts сохраняется как есть (подсветка — задача UI, У-3).
- Cypher-шаблоны: минимум под 4 эталонных запроса + сравнительный «RU vs зарубеж»
  (У-2) + gap-матрица NOT EXISTS (для analytics). Слоты — только параметризованные
  ($param), никакой конкатенации строк.
