# Паспорт модуля: api (Could — C-01, не на критическом пути)

**Назначение:** тонкий FastAPI-фасад поверх search/analytics — для «развёрнутого
решения», которое жюри может потрогать HTTP-запросами, и как задел расширяемости.

**Вход:** HTTP (JSON). **Выход:** JSON-сериализация моделей contracts
(Answer, GapReport, LitReview).

**Контракты:** те же модели `contracts.py`; API-слой своих структур не заводит,
эндпоинт = функция search/analytics + сериализация.

**Зависимости:** fastapi, uvicorn; search, analytics.

**Не входит в зону ответственности:** бизнес-логика, состояние, авторизация
(RBAC-заглушка — C-02, отдельно).

**Известные ограничения:** UI работает БЕЗ этого модуля (импортирует search напрямую) —
api не должен становиться зависимостью критического пути. Браться только по явной
задаче из STATUS (C-01) при запасе времени.
