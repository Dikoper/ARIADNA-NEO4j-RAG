# Паспорт модуля: analytics

**Назначение:** аналитика поверх готового графа: карта пробелов ⭐ (главная демо-фича),
литобзор с консенсусом/разногласиями, таблица сравнения технологий (У-2),
блок «Рекомендации» (У-1), экспорт MD/JSON-LD.

**Вход:** наполненный Neo4j (только чтение) + векторный индекс (для похожих кейсов).
**Выход:** GapReport, LitReview (с ComparisonRow), list[Recommendation],
файлы экспорта `.md` / `.jsonld`.

**Контракты:** GapReport, GapCell, LitReview, ReviewSection, ComparisonRow,
Recommendation, RecommendationKind (`contracts.py`).

**Зависимости:** Cypher-шаблоны из graph (gap-матрица NOT EXISTS, обходы);
search/embeddings для векторной близости (похожие кейсы); Claude API для
текстового синтеза секций литобзора.

**Не входит в зону ответственности:** запись в Neo4j, ответы на прямые вопросы
(search), отрисовка (ui).

**Известные ограничения:**
- Карта пробелов: комбинации «материал × процесс × условие» без источников
  (n_sources=0) + темы only_ru/only_foreign. Пример из задания — «холодный климат +
  кучное выщелачивание + никелевая руда» — должен находиться.
- Рекомендации (У-1) дёшевы по построению: похожие кейсы = top-k векторной близости;
  эксперты = обход Expert—described_in—Publication по теме; смежные темы = соседние
  узлы онтологии. Не изобретать сложнее.
- Таблица сравнения (У-2) собирается из тех же данных, что литобзор
  (метод/параметры/география/год) — не отдельный пайплайн.
