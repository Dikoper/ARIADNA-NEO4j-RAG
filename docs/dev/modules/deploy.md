# Паспорт модуля: deploy

**Назначение:** инфраструктура одного запуска: docker-compose (Neo4j + Ollama),
переменные окружения, инструкция «поднять всё с нуля» для жюри и для агентов.

**Вход:** `.env` (по образцу `.env.example`).
**Выход:** `deploy/docker-compose.yml` (+ при необходимости Dockerfile приложения),
раздел «Быстрый старт» в README.

**Контракты:** не использует (инфраструктура).

**Зависимости:** docker / docker compose на хосте; образы neo4j (с APOC и vector
index — Neo4j ≥ 5.x), ollama/ollama.

**Не входит в зону ответственности:** код пайплайна, наполнение графа, модели Ollama
(pull модели — команда в инструкции, не в образе).

**Известные ограничения:**
- Железо — DGX Spark (ARM64, GB10): образы должны быть multi-arch (проверить
  linux/arm64!), Ollama использует GPU хоста.
- Healthcheck обоих сервисов обязателен (DEPLOY-001): пайплайн стартует только
  после готовности Neo4j.
- Volumes: данные Neo4j и модели Ollama — в именованных volume, не в git.
- Пароли/ключи — только из `.env`; `.env` в git не попадает (уже в .gitignore).
