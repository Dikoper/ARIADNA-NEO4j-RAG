# Паспорт модуля: deploy

**Назначение:** инфраструктура одного запуска: docker-compose (Neo4j + vLLM + Ollama),
переменные окружения, инструкция «поднять всё с нуля» для жюри и для агентов.

**Вход:** `.env` (по образцу `.env.example`).
**Выход:** `deploy/docker-compose.yml` (+ при необходимости Dockerfile приложения),
раздел «Быстрый старт» в README.

**Контракты:** не использует (инфраструктура).

**Зависимости:** docker / docker compose на хосте; образы neo4j (с APOC и vector
index — Neo4j ≥ 5.x), vllm (NVFP4 на Blackwell), ollama/ollama.

**Не входит в зону ответственности:** код пайплайна, наполнение графа, скачивание
весов моделей (pull/download — команды в инструкции, не в образе).

**Известные ограничения:**
- Железо — DGX Spark (ARM64, GB10/Blackwell): образы должны быть multi-arch
  (проверить linux/arm64!), vLLM и Ollama используют GPU хоста.
- **Извлечение — основной рантайм vLLM + NVFP4-сборка Qwen3.5-35B-A3B** (решение PM
  03.07.2026: ~600 ток/с в батче против ~40 у Ollama Q4 — критично для дедлайна).
  Точный HF-id сборки уточнить при реализации (есть готовые рецепты vLLM для
  DGX Spark, напр. RedHatAI NVFP4); OpenAI-совместимый эндпоинт на VLLM_BASE_URL.
- **Ollama — два назначения:** эмбеддинги (qwen3-embedding:0.6b) и запасной рантайм
  извлечения (qwen3.5:9b), если vLLM/NVFP4 не заведётся за разумный тайм-бокс.
- Healthcheck всех сервисов обязателен (DEPLOY-001): пайплайн стартует только
  после готовности Neo4j.
- Volumes: данные Neo4j и веса моделей — в именованных volume, не в git.
- Пароли/ключи — только из `.env`; `.env` в git не попадает (уже в .gitignore).
