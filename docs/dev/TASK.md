# TASK.md — задание хакатона (выжимка для агентов)

Задача «Научный клубок»: единая **карта знаний R&D для горно-металлургической отрасли**.
Из разнородных документов (статьи, обзоры, отчёты, доклады, RU/EN) извлекаются сущности
и связи, строится граф знаний, система отвечает на естественно-языковые запросы
с числовыми ограничениями, географией, временными рамками и провенансом до источника.

## Онтология (фиксирована заданием)

- **8 типов сущностей:** Material, Process, Equipment, Property, Experiment,
  Publication, Expert, Facility.
- **6 типов связей:** uses_material, operates_at_condition, produces_output,
  described_in, validated_by, contradicts.

Enum-ы — в `src/ariadna/contracts.py` (EntityType, RelationType). Не расширять без solver.

## 4 эталонных запроса = приёмочные тесты решения

1. «Какие методы обессоливания воды подходят для обогатительной фабрики, если исходная
   вода содержит сульфаты, хлориды, Ca, Mg, Na по 200–300 мг/л, а требуемый сухой
   остаток — ≤1000 мг/дм³?»
2. «Какие технические решения организации циркуляции католита при электроэкстракции
   никеля описаны в мировой практике, и какая скорость потока считается оптимальной?»
3. «Покажите все эксперименты и публикации по распределению Au, Ag и МПГ между
   медным/никелевым штейном и шлаком за последние 5 лет».
4. «Какие способы закачки шахтных вод в глубокие горизонты применялись в России
   и за рубежом, и каковы их технико-экономические показатели?»

Каждый модуль пишется так, чтобы эти запросы проходили сквозной путь:
вопрос → роутер → граф+вектор → ответ с цитатами → подграф в UI.

## Ключевые требования задания

- Многопараметрические запросы: материал + процесс + условия + география + период.
- **Числовые ограничения и диапазоны** («сульфаты ≤300 мг/л») — ошибки в извлечении
  концентраций/температур недопустимы → числа и единицы извлекаются ТОЛЬКО правилами.
- Верификация знаний: источник, уровень достоверности, дата актуализации (provenance).
- Различение отечественной и зарубежной практики (гео-фильтр, сравнительные запросы).
- Синонимия RU/EN: «электроэкстракция»/«electrowinning», «ПВП»/«печь взвешенной плавки».
- Визуализация: цепочки «материал → процесс → оборудование → результат», подсветка
  противоречий (contradicts) и **пробелов** («нет экспериментов для комбинации:
  холодный климат + кучное выщелачивание + никелевая руда»).
- Литобзор: группировка источников, консенсус/разногласия, степень уверенности.
- Рекомендации: похожие кейсы, эксперты по теме, смежные темы.
- Нефункциональные: интуитивный UI (запрос на естественном языке), ответ 3–5 с,
  надёжная обработка ошибок и детальное логирование, модульная расширяемость.

## Приоритеты (MoSCoW)

- **Must:** инфраструктура (docker-compose) → ingest → векторный слой (страховочный RAG)
  → извлечение (LLM по онтологии + правила чисел) → граф с двумя хабами (Experiment,
  TechSolution/Process) и provenance → роутер с шаблонными Cypher → ответ с цитатами →
  UI: чат + подграф + карта пробелов.
- **Should:** литобзор; фильтры география/год; синонимия RU/EN; блок «Рекомендации» (У-1);
  сравнительный режим «RU vs зарубеж» + таблица сравнения технологий (У-2); подсветка
  contradicts (У-3); версионирование-минимум `source/updated_at/confidence` + ручная
  правка экспертом через Neo4j Browser (У-4).
- **Could:** RBAC-заглушка (2–3 роли), дашборд руководителя, распаковка архивов,
  PDF-экспорт.
- **Won't (в рамках хакатона):** OCR сканов, полное извлечение по всем 1453 файлам,
  продовый RBAC/аудит, уведомления, производительность на 1 млн сущностей
  (закрывается слайдом об архитектуре).
