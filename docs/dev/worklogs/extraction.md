## 2026-07-03 20:50 · module-dev (Sonnet) · A-07
**Сделано:** `src/ariadna/extraction/__init__.py`, `units.py` (таблицы UNIT_TABLE/
OPERATOR_RULES/PARAM_KEYWORDS + скомпилированные regex NUMBER_RE_STR/RANGE_*/SINGLE_RE),
`rules.py` (`extract_constraints(text) -> list[NumericConstraint]`, чистая функция,
только re). Покрыты единицы мг/л, мг/дм³, г/л, г/дм³, °C/градусы, м³/ч, м3/ч, т/сут, %;
операторы LE/LT/GE/GT/EQ/RANGE (слова + явные знаки <,>,≤,≥); числа с запятой/точкой
и разрядами через пробел/НБП; диапазоны дефис/тире/от-до с приоритетом над одиночными.
**Решения:** канон концентраций — мг/л (г/л, г/дм³ ×1000; мг/дм³ = мг/л ×1; прочие
единицы канон = унифицированное написание). Для RANGE `value`/`value_max` — сырые
(как в тексте), `norm_value` — только нижняя граница в каноне (в contracts.py нет
norm_value_max) — потребитель A-10 при необходимости конвертирует value_max тем же
множителем unit→norm_unit. Constraint эмиттится только при найденной рядом единице
(без единицы `norm_unit` нечем заполнить осмысленно) — «голые» числа (годы, номера)
игнорируются. Оператор/параметр — эвристика по ближайшей слева фразе в окне
30/60 симв., обрезаемом по границе предложения; без оператора — EQ по умолчанию;
без параметра — `param=""` (constraint всё равно возвращается).
**Проблемы:** нет.
**Открыто:** param для перечислений («сульфаты, хлориды, Ca, Mg, Na по 200–300 мг/л»)
берёт ближайшее слово (Na) — не разворачивает список; параметр справа от числа
(«от 200 до 300 мг/л сульфатов») не подхватывается (эвристика смотрит только влево);
отрицательные числа не поддержаны (минус неотличим от тире диапазона). Примеры
вход→выход (смоук): «минерализация до 10 000 мг/л» → param=минерализация, LE,
10000.0 мг/л; «температура не ниже 60 °C» → param=температура, GE, 60.0 °C;
«скорость потока 200–300 м³/ч» → param=скорость потока, RANGE, 200.0..300.0 м³/ч.
